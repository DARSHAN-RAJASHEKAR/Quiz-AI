# AI Quiz Generator 🧠

Production-grade AI-powered quiz generation platform. Upload a PDF, paste text, or describe a topic — get MCQs, true/false, and short-answer questions generated by Claude in seconds.

## Architecture

```
┌─────────────────┐    ┌─────────────────┐    ┌──────────────────┐
│   React + Vite  │───▶│   FastAPI       │───▶│   RabbitMQ       │
│   (TypeScript)  │    │   REST API      │    │   (aio-pika)     │
└─────────────────┘    └─────────────────┘    └──────────────────┘
        │ SSE                  │                        │
        │◀─────────────────────│                        ▼
        │              ┌───────┴────────┐    ┌──────────────────┐
        │              │  PostgreSQL    │◀───│  Python Worker   │
        │              │  (SQLAlchemy)  │    │  (Claude API)    │
        │              └───────────────┘    └──────────────────┘
```

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React 18 + TypeScript + Vite + Tailwind CSS v4 |
| Backend | FastAPI (Python 3.11) + SQLAlchemy async + Alembic |
| Queue | RabbitMQ 3.13 via aio-pika |
| AI Worker | Python + Anthropic Claude API (`claude-opus-4-7`) |
| Database | PostgreSQL 16 |
| Auth | JWT (python-jose + passlib/bcrypt) |
| Infra | Docker Compose |

## Quick Start

### 1. Clone and configure

```bash
git clone <repo>
cd ai-quiz
cp .env.example .env
```

Edit `.env` and set:
```bash
ANTHROPIC_API_KEY=sk-ant-...          # Required: your Claude API key
SECRET_KEY=$(openssl rand -hex 32)    # Already generated if you ran setup
```

### 2. Start all services

```bash
docker compose up --build
```

This starts:
- **PostgreSQL** on port `5432`
- **RabbitMQ** on ports `5672` (AMQP) and `15672` (Management UI)
- **FastAPI backend** on port `8000`
- **Python AI worker** (2 replicas)
- **React frontend** on port `5173`

### 3. Run database migrations

```bash
docker compose exec backend alembic upgrade head
```

### 4. Open the app

- **Frontend**: http://localhost:5173
- **API docs**: http://localhost:8000/docs
- **RabbitMQ UI**: http://localhost:15672 (user/pass from `.env`)

## API Reference

All endpoints are prefixed with `/api/v1`. Full docs at http://localhost:8000/docs.

### Auth
| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/auth/register` | Create account |
| `POST` | `/auth/login` | Login → JWT tokens |
| `GET` | `/auth/me` | Current user |
| `POST` | `/auth/refresh` | Refresh access token |

### Quizzes
| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/quizzes` | Create quiz (multipart for PDF, form for text/topic) |
| `GET` | `/quizzes` | List quizzes (paginated) |
| `GET` | `/quizzes/{id}` | Get quiz + questions |
| `DELETE` | `/quizzes/{id}` | Delete quiz |

### Jobs
| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/jobs/{id}/status/stream` | SSE stream (pass `?token=`) |
| `GET` | `/jobs/{id}/status` | Polling fallback |

### Attempts
| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/attempts` | Submit quiz attempt → score |
| `GET` | `/attempts` | List attempts |
| `GET` | `/attempts/{id}` | Get attempt detail |

## Quiz Generation Flow

```
User submits form
      │
      ▼
POST /api/v1/quizzes
      │
      ├─ Validate input / save PDF
      ├─ Insert quiz record (status=pending)
      └─ Publish message to RabbitMQ quiz.generation queue
             │
             ▼
        Worker picks up message (prefetch_count=1)
             │
             ├─ status → processing
             ├─ Extract text (PDF/text/topic)
             ├─ Call Claude API with type-specific prompt
             ├─ Parse JSON response
             ├─ Save questions to DB
             └─ status → completed
                    │
                    ▼
              SSE stream notifies browser → auto-redirect to quiz
```

## Development

### Backend only (without Docker)

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
# Ensure postgres + rabbitmq are running
alembic upgrade head
uvicorn app.main:app --reload
```

### Worker only

```bash
cd worker
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
ANTHROPIC_API_KEY=sk-ant-... python -m app.main
```

### Frontend only

```bash
cd frontend
npm install
VITE_API_URL=http://localhost:8000 npm run dev
```

## Environment Variables

See [.env.example](.env.example) for all variables.

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | ✅ | Claude API key |
| `SECRET_KEY` | ✅ | JWT signing secret (32+ bytes hex) |
| `POSTGRES_*` | ✅ | Database credentials |
| `RABBITMQ_*` | ✅ | RabbitMQ credentials |
| `MAX_UPLOAD_SIZE_MB` | — | PDF upload limit (default: 20) |

## Scaling

The worker is stateless and can scale independently:

```bash
docker compose up --scale worker=4
```

RabbitMQ's `prefetch_count=1` ensures fair distribution across all worker replicas.
